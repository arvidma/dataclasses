# make_dataclass(decorator=...) ignores all parameters

## Severity: HIGH

## Description

When `make_dataclass()` is called with a custom `decorator`, all
configuration parameters (`init`, `repr`, `eq`, `order`, `unsafe_hash`,
`frozen`, `match_args`, `kw_only`, `slots`, `weakref_slot`) are silently
ignored. They are never forwarded to the decorator.

## Original code (dataclasses.py)

```python
if decorator is None:
    return dataclass(
        cls, init=init, repr=repr, eq=eq, order=order,
        unsafe_hash=unsafe_hash, frozen=frozen,
        match_args=match_args, kw_only=kw_only,
        slots=slots, weakref_slot=weakref_slot,
    )
return decorator(cls)  # <-- ALL keyword arguments DISCARDED
```

## CPython 3.14 (reference)

CPython 3.14 always forwards all parameters, regardless of whether the
default decorator or a custom one is used:

```python
cls = decorator(cls, init=init, repr=repr, eq=eq, order=order,
                unsafe_hash=unsafe_hash, frozen=frozen,
                match_args=match_args, kw_only=kw_only, slots=slots,
                weakref_slot=weakref_slot)
```

## Impact

A user writing:

```python
make_dataclass('C', ['x'], frozen=True, decorator=my_decorator)
```

expects `frozen=True` to reach `my_decorator`. With the old code,
`my_decorator(cls)` was called without the `frozen` argument.

## Sources

- GitHub Issue: https://github.com/python/cpython/issues/118974
- GitHub PR: https://github.com/python/cpython/pull/122723
- CPython main branch `Lib/dataclasses.py`, `make_dataclass()`

## Resolution

Fixed by always forwarding all parameters to the decorator, matching
CPython 3.14 behavior:

```python
if decorator is None:
    decorator = dataclass
return decorator(
    cls, init=init, repr=repr, eq=eq, order=order,
    unsafe_hash=unsafe_hash, frozen=frozen,
    match_args=match_args, kw_only=kw_only,
    slots=slots, weakref_slot=weakref_slot,
)
```
