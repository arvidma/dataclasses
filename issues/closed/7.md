# Redundant default_factory check in replace()

## Severity: INFO

## Description

In `replace()`, the backport checked `f.default_factory is MISSING` as
part of the condition for raising ValueError on InitVar fields without
explicit values. This check was dead code because InitVar fields can
never have `default_factory` - it raises TypeError at class creation
time (in `_get_field()`).

## Original code (dataclasses.py)

```python
if f._field_type is _FIELD_INITVAR:
    if f.default is MISSING and f.default_factory is MISSING:
        raise ValueError(
            f"InitVar {f.name!r} must be specified with replace()"
        )
```

## CPython (all versions with this fix)

CPython only checks `f.default is MISSING`:

```python
if f._field_type is _FIELD_INITVAR and f.default is MISSING:
    raise ValueError(...)
```

## Sources

- CPython 3.12 `Lib/dataclasses.py`, `replace()`
- CPython Issue: https://github.com/python/cpython/issues/80651

## Resolution

Fixed as part of issue 4. The entire `replace()` InitVar handling was
rewritten to match CPython's structure exactly.
