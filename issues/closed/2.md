# _create_fn() missing closure-wrapping

## Severity: MEDIUM

## Description

The backport's `_create_fn()` used the old simple `exec` pattern from
Python 3.7. Since CPython 3.11, `_create_fn()` uses closure-wrapping to
ensure that generated functions close over local variables properly
instead of sharing a mutable `locals` dict.

## Original code (dataclasses.py)

```python
def _create_fn(name, args, body, *, globals=None, locals=None, return_type=MISSING):
    if locals is None:
        locals = {}
    return_annotation = ""
    if return_type is not MISSING:
        locals["_return_type"] = return_type
        return_annotation = "->_return_type"
    args = ",".join(args)
    body = "\n".join(f" {b}" for b in body)
    txt = f"def {name}({args}){return_annotation}:\n{body}"
    exec(txt, globals, locals)
    return locals[name]
```

## CPython 3.12 (reference)

```python
def _create_fn(name, args, body, *, globals=None, locals=None, return_type=MISSING):
    if locals is None:
        locals = {}
    return_annotation = ''
    if return_type is not MISSING:
        locals['__dataclass_return_type__'] = return_type
        return_annotation = '->__dataclass_return_type__'
    args = ','.join(args)
    body = '\n'.join(f'  {b}' for b in body)
    txt = f' def {name}({args}){return_annotation}:\n{body}'
    local_vars = ', '.join(locals.keys())
    txt = f"def __create_fn__({local_vars}):\n{txt}\n return {name}"
    ns = {}
    exec(txt, globals, ns)
    return ns['__create_fn__'](**locals)
```

## Impact

Without closure-wrapping, generated functions hold a reference to the
mutable `locals` dict. If `locals` is modified after the function is
created, the generated function may see stale or incorrect values for
its free variables. Closure-wrapping converts locals into function
parameters, which are bound at call time and cannot be mutated
afterwards.

## Sources

- CPython 3.11 `Lib/dataclasses.py`, `_create_fn()`
- CPython 3.12 `Lib/dataclasses.py`, `_create_fn()`
- gh-96151: Rename internal variable names to avoid shadowing with field names

## Resolution

Updated `_create_fn()` to use the closure-wrapping pattern from
CPython 3.12. The generated function is now wrapped in a
`__create_fn__()` factory whose parameters are the local variables.
This also renames `_return_type` to `__dataclass_return_type__` to
match CPython 3.12's naming (this was issue 3).
